\section{Thiết kế custom actions}

Custom Action là thành phần cho phép Rasa thực hiện các xử lý ngoài khả năng trả lời tĩnh, ví dụ gọi Backend API, truy vấn cơ sở dữ liệu, gọi phân hệ RAG hoặc xử lý logic nghiệp vụ.

Trong hệ thống chatbot tư vấn tuyển sinh, Custom Action được đặt trong Action Server. Khi Rasa Core dự đoán cần thực hiện một action, Rasa Server sẽ gửi request sang Action Server. Action Server xử lý logic và trả về kết quả cho Rasa để phản hồi người dùng.

\subsection*{Luồng xử lý Custom Action}

Luồng xử lý tổng quát của một Custom Action gồm các bước:
\begin{enumerate}
    \item Nhận intent, entities, slots và câu hỏi gốc từ Rasa.
    \item Kiểm tra dữ liệu đầu vào.
    \item Nếu thiếu thông tin quan trọng, yêu cầu người dùng bổ sung.
    \item Nếu đủ thông tin, gọi Backend API hoặc RAG service.
    \item Nhận kết quả trả về.
    \item Định dạng câu trả lời.
    \item Gửi phản hồi cho người dùng.
    \item Cập nhật slot nếu cần.
\end{enumerate}

\subsection*{Ví dụ action lấy điểm chuẩn}

\begin{verbatim}
class ActionGetBenchmarkScore(Action):

    def name(self) -> Text:
        return "action_get_benchmark_score"

    def run(
        self,
        dispatcher: CollectingDispatcher,
        tracker: Tracker,
        domain: Dict[Text, Any]
    ) -> List[Dict[Text, Any]]:

        major = tracker.get_slot("major")
        year = tracker.get_slot("year")

        if not major:
            dispatcher.utter_message(
                text="Bạn muốn hỏi điểm chuẩn của ngành nào?"
            )
            return []

        if not year:
            year = "2024"

        response = requests.get(
            f"{BACKEND_API_URL}/admissions/benchmark-score",
            params={
                "major": major,
                "year": year
            }
        )

        if response.status_code != 200:
            dispatcher.utter_message(
                text="Hiện tại mình chưa lấy được thông tin điểm chuẩn. Bạn vui lòng thử lại sau."
            )
            return []

        data = response.json()

        if not data:
            dispatcher.utter_message(
                text=f"Mình chưa tìm thấy điểm chuẩn ngành {major} năm {year}."
            )
            return []

        message = (
            f"Điểm chuẩn ngành {major} năm {year} là {data['score']} điểm. "
            f"Phương thức xét tuyển: {data.get('method', 'chưa có thông tin')}."
        )

        dispatcher.utter_message(text=message)

        return [
            SlotSet("last_answer_type", "dynamic")
        ]
\end{verbatim}

\subsection*{Thiết kế nguyên tắc cho Custom Action}

Custom Action không nên chứa quá nhiều logic phức tạp về dữ liệu. Các nghiệp vụ chính như truy vấn ngành, học phí, điểm chuẩn nên được đặt ở Backend API. Action Server chỉ đóng vai trò trung gian giữa Rasa và Backend.

Cách thiết kế này giúp hệ thống dễ bảo trì hơn. Nếu dữ liệu tuyển sinh thay đổi, chỉ cần cập nhật Backend API hoặc cơ sở dữ liệu, không cần chỉnh sửa trực tiếp logic của chatbot.
