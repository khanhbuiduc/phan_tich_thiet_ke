\section{Thiết kế tổng quan phân hệ RAG}

\subsection{Kiến trúc tổng quan của phân hệ RAG}

Phân hệ RAG được thiết kế như một dịch vụ độc lập, có nhiệm vụ tiếp nhận tài liệu tuyển sinh, xử lý tài liệu, lập chỉ mục tri thức và cung cấp API truy vấn cho các thành phần khác như Rasa, Backend API và phân hệ quản trị.

Các nhóm thành phần chính gồm:
\begin{itemize}
    \item Data Ingest Pipeline.
    \item Preprocessing Module.
    \item Embedding Module.
    \item Storage Layer.
    \item Retrieval Module và Reranking Module.
    \item Context Builder và Prompt Assembly.
    \item Generation Module và Citation Module.
    \item Monitoring and Logging.
\end{itemize}

\subsection{Quan hệ giữa RAG, Rasa, Backend API và Admin}

Rasa tiếp nhận tin nhắn người dùng, nhận diện intent và điều phối hội thoại. Khi câu hỏi thuộc nhóm cần tra cứu tài liệu, Rasa gọi Custom Action để gửi truy vấn đến RAG Service.

Backend API là lớp trung gian quản lý dữ liệu hệ thống, xác thực người dùng, lưu lịch sử hội thoại, quản lý tài liệu và cung cấp API cho frontend hoặc admin.

Admin có quyền tải tài liệu, kiểm duyệt, kích hoạt lập chỉ mục, xóa tài liệu lỗi thời và theo dõi chất lượng trả lời. RAG Service là phân hệ xử lý tri thức, tiếp nhận tài liệu từ admin, lập chỉ mục và trả lời truy vấn có căn cứ.
